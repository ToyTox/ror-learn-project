Для выполнения HTTP запросов из терминала (консоли) используйте пакет ncat: https://nmap.org/download.html - современную и более продвинутую версию netcat.

Используйте один из подходящих вариантов установки (пакет ncat входит в состав пакета nmap):

macOS: brew install nmap
Ubuntu: sudo apt-get install nmap
Windows: nmap-7.60-setup.exe
или инструкции на официальной странице: https://nmap.org/book/install.html

Для выполнения запросов из задания используйте этот URL сервиса http://httpbin.org/:

http://httpbin.org/anything

При этом, в теле ответа параметры строки запроса будут находиться в свойстве args, параметры POST запроса - в свойстве data.

Внимание!

При вызове ncat обязательно передайте опцию -C (преобразует символ новой строки в CRLF - возврат каретки и символ новой строки, этого требует спецификация HTTP). Также спецификация HTTP/1.1 требует обязательного наличия заголовка запроса Host. В данном случае он должен иметь вид:
Host: httpbin.org
POST запрос требует наличия заголовка Content-Length
Задание.

1. Сформируйте и отправьте по указанному выше url:

GET запрос без параметров
GET запрос с любыми параметрами строки URL
POST запрос с любыми параметрами
Сформированные запросы и полученные ответы скопируйте в сервис https://gist.github.com/ и выложите ссылку на полученный листинг в ответе на задание

2. Перечислите в ответе на задание HTTP-методы для создания/получения/обновления/удаления ресурса

3. Объясните в ответе на задание, что делают данные заголовки: Accept, Connection, Content-Type, User-Agent, Content-Length, Cache-Control и где они могут встречаться (в заголовках запроса, ответа, в обоих случаях)
